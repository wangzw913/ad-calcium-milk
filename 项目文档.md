# AD钙奶·酸甜成长记 — H5互动游戏 项目文档

## 一、项目概述

### 1.1 项目背景
**赛事**：第18届全国大学生广告艺术大赛（大广赛）  
**命题品牌**：娃哈哈  
**命题产品**：AD钙奶（1996年上市，2026年恰逢30周年）  
**广告主题**：二选一 ——「陪你酸酸甜甜 懂你可可爱爱」/「不管几岁 快乐万岁」  
**作品赛道**：互动类 → 线上互动（H5）  
**作品名称**：AD钙奶 · 酸甜成长记  

### 1.2 核心创意
人生有酸有甜，AD钙奶始终陪伴。本作品是一款**移动端H5互动收集小游戏**，玩家拖动角色接住从天空掉落的AD钙奶各口味产品和酸甜记忆，在30秒内经历「童年→青春→现在」三个成长阶段，最终获得专属「AD人格」标签。

**主题诠释**：
- **「陪你酸酸甜甜」** — 酸（柠檬）和甜（AD钙奶）都会掉落，都要接住，象征成长中的苦与乐
- **「懂你可可爱爱」** — 卡通风美术、动漫人物、萌系音效，传递品牌「童心童趣」调性

### 1.3 技术栈
| 技术 | 用途 |
|------|------|
| HTML5 | 页面结构 |
| CSS3 | 动效与布局（毛玻璃、动画、过渡） |
| Canvas 2D API | 游戏核心渲染（背景、人物、掉落物、粒子） |
| Web Audio API | 背景音乐循环 + 音效反馈 |
| Node.js (sharp) | 图片处理（去白底、裁剪透明边距） |
| Git + GitHub Pages | 版本管理 + 线上部署 |

## 二、架构设计

### 2.1 页面结构（四屏模式）

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  开始页       │───▶│  倒计时       │───▶│  游戏页       │───▶│  结算页       │
│  (设计稿背景) │    │  (3-2-1-GO)  │    │  (30秒)      │    │  (设计稿背景) │
│  + 开始按钮   │    │  + 粒子特效   │    │  + 三阶段背景 │    │  + 分数卡片   │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
```

页面切换使用 CSS `opacity` 过渡 + `pointer-events` 控制，所有页面绝对定位叠加，通过 `.hid` 类切换可见性。

### 2.2 游戏核心架构（Canvas）

```
gameLoop(timestamp)
  ├── update(dt)
  │     ├── 计时器递减 + 紧急动画
  │     ├── 阶段切换检测 (0-10s童年 / 10-20s青春 / 20-30s现在)
  │     ├── 玩家位置平滑跟随 (lerp插值)
  │     ├── 云朵漂移
  │     ├── 物品生成计时器
  │     ├── 物品下落 + 碰撞检测
  │     ├── 粒子更新
  │     └── 浮动文字更新
  └── render()
        ├── 绘制阶段背景 (设计稿图片)
        ├── 绘制云朵
        ├── 绘制掉落物品 (产品PNG / 柠檬 / 爱心)
        ├── 绘制粒子
        ├── 绘制浮动文字
        ├── 绘制角色 (用户提供的三阶段人物图)
        └── 绘制光晕 + 阶段过渡效果
```

### 2.3 数据流
- **输入**：触摸/鼠标事件 → 屏幕坐标 → `targetX`
- **状态**：`playerX`（lerp跟随 `targetX`）、`items[]`、`particles[]`、`floatTexts[]`
- **输出**：Canvas 每帧重绘全部元素

## 三、详细实现

### 3.1 开始页
使用用户设计的 `开始页.png` 作为全屏背景（`object-fit: cover`），叠加：

- **闪烁装饰**：8个CSS动画星星元素（✨💚🌸💫🌟🧃）
- **开始按钮**：半透明毛玻璃样式 + 呼吸光晕动画，位于页面底部16%处

### 3.2 倒计时系统

- **数字**：娃哈哈绿色 `#4CAF50`，缩放弹出动画
- **粒子**：每次数字跳动触发12个彩色粒子向四周爆发 + 脉冲光环
- **GO!**：橙色大字 + 双倍粒子 + 启动游戏循环 + 背景音乐

- 缩放范围：0.25（童年）→ 0.30（青春）→ 0.33（现在），体现成长
- 位置：基于地面线 `gnd + 40px`，确保踩地不悬浮
- 阴影：Canvas 手绘椭圆阴影增强立体感

- **音量控制**：BGM增益0.18，单音0.12，避免刺耳

## 四、关键技术难点与解决方案

### 4.1 坐标系不一致导致碰撞失效
**问题**：物品X坐标使用屏幕像素（`gW`），人物X坐标使用游戏坐标系（`GW=430`），两者不在同一尺度，导致碰撞检测永远无法匹配。

**解决**：统一所有坐标到屏幕像素空间。修改触摸事件处理器 `getCP()` 直接返回屏幕坐标（不再缩放到GW空间），物品生成也使用 `gW`。碰撞检测从此正常工作。

### 4.2 Logo白底问题
**问题**：从AI/PDF转换的Logo PNG有白色背景，放在游戏/结算页上显得突兀。

**解决**：使用 Node.js `sharp` 库逐像素处理——遍历所有像素，将RGB值>240的白色/近白色像素的Alpha通道设为0（透明）。再使用 `.trim()` 裁剪透明边距，确保Logo主体紧贴图片边缘。处理后Logo在任意背景上都干净透明。

### 4.3 人物图白底与尺寸
**问题**：用户提供的三张人物图同样有白色背景，且原始尺寸过大超出屏幕。

**解决**：同样用 `sharp` 去白底+裁剪。缩放系数从最初的0.55逐步调整到0.25，起始scale从1.2调整到0.25，经过反复微调找到合适大小。

### 4.4 掉落物品变形
**问题**：硬编码 `width=70, height=126` 的固定宽高比导致不同产品图拉伸变形。

**解决**：改为动态计算——读取每张图片的原始宽高（`naturalWidth/naturalHeight`），按原始比例计算渲染宽度：`iw = ih * (natW/natH)`。

### 4.5 人物悬浮感
**问题**：人物看起来悬空，脚不踩地。

**解决**：调整 `drawCharacter` 的图片锚点——将 `drawImage` 的Y偏移从 `-0.85*h` 改为 `-0.95*h`，使图片底部对齐地面线。阴影也同步调整到脚底位置。

### 4.6 开始按钮遮挡设计稿
**问题**：开始按钮覆盖了设计稿中的小男孩脚部。

**解决**：按钮从 `bottom:5%` 逐步上移至 `bottom:16%`，同时改为半透明毛玻璃样式（`rgba(76,175,80,.6)` + `backdrop-filter: blur(6px)`），不遮挡主体内容。

### 4.7 页面不连贯
**问题**：用户反馈加载动画与整体设计风格不协调。

**解决**：删除独立的加载页，改为「开始页 → 倒计时 → 游戏页 → 结算页」直通流程。页面间使用CSS `opacity 0.45s` 过渡，视觉连贯。

### 4.8 GitHub部署网络问题
**问题**：HTTPS 推送被阻断（Connection reset），GitHub API 认证超时（TLS handshake timeout）。

**解决**：
1. 安装 GitHub CLI 尝试web认证 → API被阻断
2. 尝试HTTPS推送 → 端口443被reset
3. 尝试标准SSH（端口22） → 同样被reset
4. **最终方案**：配置SSH走443端口（`ssh.github.com:443`），生成ED25519密钥对，用户添加公钥到GitHub → 推送成功

### 4.9 结算页文字可读性
**问题**：用户设计稿色彩丰富，分数文字叠加在上面难以辨认。

**解决**：经过多轮调整（透明度从0.9→0.55→0.35→0.2→0.1→0.06），最终采用极淡的白色底（`rgba(255,255,255,.06)`）+ 深色文字，同时加深阴影和描边保证可读性。

### 4.10 顶栏排版冲突
**问题**：游戏顶栏Logo、阶段标签、分数、倒计时拥挤冲突。

**解决**：采用暗色毛玻璃统一风格（`rgba(0,0,0,.5)` + `backdrop-filter: blur(12px)`），阶段标签和分数胶囊居中排列，倒计时独立圆形靠右，Logo独立左上角，层级分明。

## 六、设计理念总结

### 6.1 品牌契合度
- AD钙奶调性「童心、童趣、可爱、快乐」贯穿全作：明亮色系、Q萌人物、轻快BGM
- 「酸甜」概念具象化为游戏机制：甜（产品瓶加分多）和酸（柠檬分低但不扣分）都要接住
- 「陪伴成长」映射为三阶段背景+人物成长+主题语变化

### 6.2 用户体验设计
- **零学习成本**：单指拖动，直观操作
- **即时反馈**：接住物品 → 粒子爆炸 + 浮动数字 + 清脆音效
- **情感连接**：30秒浓缩「童年→青春→现在」成长旅程，引发共鸣
- **社交传播**：个性AD人格标签 + 一键分享文案

### 6.3 技术选型理由
- **纯前端**：单HTML文件，无框架依赖，GitHub Pages即可部署
- **Canvas渲染**：60fps流畅动画，支持大量掉落物品同时渲染
- **Web Audio**：无需音频文件，合成音效体积为零
- **响应式**：适配所有手机屏幕，`max-width:430px` + `dvh` + `safe-area-inset`

## 七、面试可能被问到的问题

**Q: 为什么选择H5而不是小程序或其他形式？**
A: 大广赛互动类明确包含H5赛道；H5无需安装、扫码即玩、分享方便；单文件部署成本低。

**Q: 碰撞检测是怎么实现的？**
A: 矩形区域判定——物品Y进入 `gnd-60` 到 `gnd+90` 范围且X偏差小于120px即判定碰撞。比圆形距离判定更宽容，用户体验更好。

**Q: 三阶段切换怎么做的？**
A: `update()` 每帧根据 `timeLeft` 计算当前阶段（0-10s→阶段0，10-20s→阶段1，20-30s→阶段2），检测到变化时触发标语弹出、音效、进度条更新、背景图切换。

**Q: 人物成长如何表现？**
A: 三张不同年龄的人物图 + 动态缩放（`charScale=0.25+stage*0.05`）+ 服装/发型/配饰变化。

**Q: 最大的技术挑战是什么？**
A: 坐标系统一问题。物品和人物使用不同坐标空间导致碰撞失效，排查了很久才发现。最终统一到屏幕像素坐标系解决。

**Q: 如何优化大图片加载？**
A: 所有资源在游戏开始前通过 `Promise.all` 预加载为Image对象，开始页展示时后台加载，用户点击时资源已就绪。后期可进一步压缩PNG或转为WebP。

**Q: 怎么部署上线的？**
A: Git推送到GitHub仓库 → 开启GitHub Pages → 自动分配 `wangzw913.github.io/ad-calcium-milk` 域名。推送遇到网络封锁，通过SSH 443端口绕过的。
